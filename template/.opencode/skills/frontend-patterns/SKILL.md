# Frontend Patterns Skill

Use this skill for all UI/frontend work.

Quality bar:
- Production-looking UI, not placeholder UI.
- Clear visual hierarchy: title, section grouping, primary action, secondary action.
- Consistent spacing scale.
- Responsive behavior for mobile, tablet, and desktop.
- Accessible semantic HTML and keyboard states.
- Loading, empty, error, success, and disabled states.
- Reuse existing components/design tokens before creating new ones.
- Avoid random colors, arbitrary shadows, inconsistent radii, and unbalanced layouts.

Implementation rules:
- Keep components small and composable.
- Separate presentation, state, and data-fetch boundaries where reasonable.
- Use form validation and helpful messages.
- Do not hardcode mock data as final behavior unless explicitly prototype mode.
